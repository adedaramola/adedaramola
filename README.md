About Me

I build production-style AI and cloud platforms that connect LLMs, AWS infrastructure, Kubernetes operations, and MLOps into practical engineering systems.

My work focuses on four core areas:

* Autonomous cloud operations: GitOps Auto-Remediation uses AI agents, confidence scoring, AWS Lambda, Step Functions, EventBridge, DynamoDB, GitHub PR automation, Argo CD, and Kubernetes to detect incidents, propose fixes, route actions by risk, and validate outcomes.
* Enterprise RAG systems: Enterprise RAG Platform is a production-style retrieval-augmented generation platform using AWS, pgvector on Aurora PostgreSQL, S3/SQS ingestion, ECS Fargate, FastAPI, Terraform, and operational runbooks.
* Multi-LLM infrastructure: Multi-LLM Platform is an AI gateway for routing requests across OpenAI, Anthropic Claude, and AWS Bedrock with cost-aware routing, semantic caching, API-key auth, rate limiting, provider health checks, observability, and Terraform-based deployment.
* LLM specialization and evaluation: LLM Specialization Platform demonstrates fine-tuning, DPO, artifact export, GGUF quantization, reproducible evaluation, regression testing, and CI-ready metrics for specialized language models.

Across these projects, I emphasize secure infrastructure, automation, observability, cost awareness, reproducibility, and honest production-readiness boundaries. My goal is to design AI systems that are not just demos, but operational platforms that teams can reason about, deploy, monitor, and improve.
